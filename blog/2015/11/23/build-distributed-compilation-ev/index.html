利用Distcc和Dmucs构建大规模分布式C++编译环境

Nov 23rd, 2015 10:48 pm

如果你的C++项目非常庞大，含有1000个.h文件，2000个.cpp文件，那么我敢打赌，每次编译所花的时间，都足够你喝3000杯咖啡了。如何加快编译速度呢？

Distcc是开源的用于搭建分布式编译环境的利器，是解决棘手问题的好帮手。然而，由于它的调度算法过于简单，不大合理，因此我们利用DMUCS提供的调度功能，来搭建一个相对完美的编译平台。

话不多说，我们的编译平台包括：

开发机（client）：这一台机器上有我们的项目、工程文件、代码，平时我们在这台机器上写代码，要编译的对象也在这台机器上。机器IP为192.168.1.99

服务器（server）：负责编译的机器，一共有3台。IP分别为192.168.1.11,192.168.1.22,192.168.1.33.

调度器（Scheduler）：负责调度的机器，把编译任务合理的派发到机器上，DMUCS就安装在这台机器上，IP为192.168.1.88

服务器

1，安装Distcc

sudo apt-get install distcc

安装后可以得到两个二进制文件，distccd和distcc。前者主要负责接收数据，后者可以认为是g++的前端。

2，安装gcc

sudo apt-get install gcc

3，配置Distcc

打开/etc/default/distcc，我们发现如下配置项：

STARTDISTCC="true"
ALLOWEDNETS="127.0.0.1 192.168.1.0/24"
LISTENER="0.0.0.0"

4，启动Distccd

sudo service distcc start

经过以上配置，每次机器开机，都会自动运行Distccd。运行如下命令确认下：

ps -aux | grep distccd

开发机

1，安装Distcc

当然啦，我们假设您的开发机上已经安装gcc了。

2，设置编译资源

这一步是指定哪些机器（也就是上面的Server）来负责编译工作。

export DISTCC_HOSTS="192.168.1.11 192.168.1.22 192.168.1.33"

/usr/local/etc/distcc/hosts

每台机器的IP之间用空格隔开。

3，应用Distcc

应用Distcc来编译代码有几种方法：

方法一：在编译的时候，直接指定使用Distcc

make -j4 CXX=distcc

这种方法，将覆盖makefile文件中CXX变量的值（一般是g++或者clang++），代之以distcc

方法二：修改makefile中的CXX的值

将makefile文件中的这一行

CXX = g++

改为

CXX = distcc

方法三：修改configure文件

如果您的makefile文件由automake产生，那么在运行./configure时得加上参数，变为：

./configure --CXX=distcc

那么生成的makefile文件将自动使用distcc了。
